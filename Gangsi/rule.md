# 古墓迷蹤（Gangsi）規則與遊戲邏輯

第 1～9 節記錄尊重原版玩法的「經典模式」。第 10 節記錄網站新增的合作「獵殺模式」；獵殺模式沿用擲骰、鎖骰、移動、抓捕與地牢核心，但覆寫勝負目標並加入固定職業與提燈怪能力。

本文件同時作為：

1. 前端「遊戲規則」面板的文案來源。
2. Server 遊戲狀態、合法行動與勝負判定的實作規格。
3. 測試案例與日後規則裁定的基準。

若本文件與原版規則的解讀不同，以「網站版裁定」為準。版圖通道應由獨立的結構化地圖資料提供，不得從背景圖或 CSS 推算。

地圖 JSON 只能描述尺寸與物件 placement，不得定義或覆寫互動規則。道路、牆壁、封閉格、入口、地牢、寶藏、冒險者與提燈怪的 class 屬性由程式固定；載入不同地圖只會改變位置，不會改變數字骰、箭頭骰、抓捕或寶藏揭露行為。

## 1. 遊戲概覽

- 遊戲人數：2 至 5 名真人玩家。
- 陣營：1 名玩家扮演提燈怪，其餘玩家扮演冒險者。
- 冒險者知道所有棋子的目前位置；提燈怪看不到冒險者的位置。
- 每名冒險者持有秘密任務卡，必須尋找卡片指定的五件寶藏。
- 冒險者彼此合作躲避提燈怪，但各自競爭，只有最先完成自己任務的冒險者獲勝。
- 提燈怪透過抓捕冒險者取得生命標記，達到門檻便獲勝。

## 2. 玩家規則

本章應作為前端規則面板的主要內容。實際畫面可以拆成分頁或摺疊段落，但不得省略會影響合法行動的規則。

### 2.1 遊戲準備

- 提燈怪不擲順序骰；所有冒險者在大廳各擲一次 d100，點數由高到低決定冒險者行動順序，同點由 Server 的隱藏亂數決定。
- 每名玩家自行選擇冒險者或提燈怪，開始時必須正好有 1 名提燈怪；已有人選擇提燈怪時，其他玩家必須等該玩家改回冒險者後才能接替。
- 每名冒險者在大廳填寫一個字作為自己的圓形棋子文字；提燈怪棋子固定顯示「怪」。
- 房主可直接選擇 catalog 地圖，或勾選隨機地圖；隨機模式只在按下開始後由 Server 抽選並向全房揭露結果。

#### 3 至 5 人遊戲

- 1 人扮演提燈怪。
- 其餘 2 至 4 人各控制一名冒險者。
- 每名冒險者有 3 點生命。
- 每名冒險者從藍、綠、黃、粉、紅五種任務牌堆各取得 1 張，共 5 張秘密任務卡。
- 所有冒險者從入口開始，提燈怪從地牢開始。

#### 2 人遊戲

- 1 人扮演提燈怪。
- 1 人控制兩名冒險者棋子，兩枚棋子各有 3 點生命。
- 冒險者玩家每種顏色取得 2 張任務卡，共 10 張，兩枚棋子共用同一手牌。
- 每輪依固定順序讓兩枚冒險者棋子各進行一個完整回合。
- 任一枚棋子都可以完成手中的任務卡。

### 2.2 回合順序

所有仍在遊戲中的冒險者依座位順序各進行一回合，之後才進行一次提燈怪正常回合。

```text
冒險者 A → 冒險者 B → 冒險者 C → 提燈怪正常回合 → 回到 A
```

每名冒險者的回合依序為：

1. 若有鎖定骰，決定是否全部解鎖。
2. 擲出目前可用的冒險者骰。
3. 視需要不限次數重新擲骰。
4. 選擇一顆骰子並完成移動。
5. 若停在任務指定的寶藏格，決定是否揭露任務卡。

選擇解鎖不會取代冒險者自己的回合。提燈怪插入回合結束後，觸發解鎖的冒險者通常會使用全部五顆骰子繼續自己的回合；若提燈怪在插入回合堵住所有合法移動，該冒險者只能按下「略過回合」。

### 2.3 冒險者骰與重擲

共有 5 顆冒險者骰，每顆六面為：

```text
1、2、3、4、箭頭、提燈怪
```

- 冒險者擲出目前所有未鎖定的骰子。
- 每次出現提燈怪面的骰子都立即鎖定，不能在本次回合繼續使用。
- 若不滿意其餘骰子的結果，可以不限次數重新擲出所有未鎖定骰。
- 重擲時不能保留某些數字骰，只重擲其他骰；每次都重擲所有未鎖定骰。
- 滿意結果後，選擇最新一次結果中的一顆骰子，向提燈怪公開所選結果，再執行移動。
- 不能選擇一個無法合法完成的結果。若目前結果都不能合法使用，必須繼續重擲。
- 進入擲骰階段後不能返回解鎖階段。
- 完成移動後，未鎖定骰恢復為下一名冒險者可使用的骰子；只有提燈怪面會持續鎖定。

#### 五顆骰子全部鎖定

- 當五顆骰子全部顯示提燈怪面，當前冒險者不能移動或揭露寶藏。
- 行動資訊公開說明五顆骰子全部鎖定，當前冒險者只能按下「略過回合」。
- 按下「略過回合」後才結束當前回合並繼續後續流程。
- 下一名能正常進行回合的冒險者必須強制解鎖，不能選擇保留五顆鎖定骰。
- 如果五顆骰子是在本輪最後一名冒險者的回合全部鎖定，先進行提燈怪正常回合，再進行一次 5 步的提燈怪插入回合，之後解鎖全部骰子。

### 2.4 解鎖與提燈怪插入回合

- 解鎖只能在冒險者回合開始、尚未擲骰前選擇。
- 解鎖時必須一次解鎖全部鎖定骰，不能只選擇部分骰子。
- 提燈怪立即取得一次插入回合，最大步數等於本次解鎖的骰子數量。
- 插入回合不擲提燈怪骰。
- 網站版允許提燈怪移動 `0～N` 步，因此可以選擇不移動或提早停止。
- 插入回合結束後歸還全部冒險者骰，並重新檢查觸發解鎖的冒險者是否仍有合法移動。
- 若仍有合法移動，該冒險者再擲五顆骰子；若已被完全封死，公開原因且只能按下「略過回合」。
- 若提燈怪在插入回合抓到觸發解鎖的冒險者，該冒險者失去 1 點生命並移到地牢；若仍未出局，他接著從地牢進行自己的擲骰與移動。

### 2.5 數字骰移動

選擇 `1、2、3、4` 時：

- 必須完整走完所選點數，不能少走或提早停止。
- 每一步只能上下左右移動，不能斜走。
- 移動途中可以轉彎。
- 可以往返，也可以在同一次移動中多次經過相同格子。
- 牆壁與提燈怪所在格不能進入或穿越。
- 可以穿越其他冒險者所在格，該格照常計算一步。
- 一般道路格最多只能停留一名冒險者，因此不能停在其他冒險者所在格。
- 進入過版圖後，不能主動走回入口或地牢。
- 若所選點數無法完整合法走完，不能選用該骰，必須重擲未鎖定骰。

範例：A 位於 `(6,3)`、B 位於 `(6,4)`。

- A 不能使用 `1`，因為會停在 B 所在格。
- A 可以使用 `2`，沿 `(6,3) → (6,4) → (6,5)` 移動。
- A 也可以使用 `2`，沿 `(6,3) → (6,4) → (6,3)` 往返。

### 2.6 箭頭骰移動

選擇箭頭時：

- 必須選擇上、下、左、右其中一個方向。
- 必須保持直線前進，不能在岔路轉彎。
- 一直移動到該方向下一個障礙物的前一格。
- 障礙物包括牆壁、版圖邊界、提燈怪、其他冒險者、入口與地牢。
- 其他冒險者對數字骰是可以穿越的棋子，但對箭頭骰是障礙物。
- 箭頭至少必須移動一格，不能原地不動。
- 若相鄰格就是障礙物，不能選擇該方向。
- 若四個方向都不能移動，箭頭不能使用。
- 途中經過寶藏不會觸發揭露，只有最後停留的格子才會檢查任務。

### 2.7 完全無法移動

若冒險者在回合開始時，無論使用 `1、2、3、4` 或箭頭都不存在任何合法移動：

- 行動資訊公開說明該冒險者沒有任何合法移動。
- 不能解鎖。
- 不能擲骰。
- 不能揭露寶藏。
- 當前冒險者只能按下「略過回合」，按下後才結束回合並輪到下一位。

只有牆壁與提燈怪可能造成真正的完全封鎖。其他冒險者可以被數字骰穿越，因此單獨堵住出口通常不會構成完全封鎖。

### 2.8 入口與地牢

入口與地牢是特殊區域，不是一般道路格。

#### 入口

- 地圖標記位置為 `(4,7)`，視覺上可以利用 `(4,7)`、`(5,7)`排列等待中的冒險者棋子。
- 所有冒險者開局都處於同一個邏輯入口狀態，可以共同等待，不受一般格單人限制。
- 從入口移動的第 1 步會進入道路格 `(3,7)`。
- 例如選擇數字 `1`，便停在 `(3,7)`。
- 離開入口後不能主動返回。

#### 地牢

- 地牢位置為 `(6,7)`。
- 提燈怪開局位於地牢；被捕且仍有生命的冒險者也會被移到地牢。
- 從地牢移動的第 1 步可以選擇進入道路格 `(6,6)` 或 `(7,7)`。
- 地牢可以同時容納多名被捕冒險者，不受一般格單人限制。
- 棋子離開地牢後不能主動返回。

### 2.9 寶藏與任務卡

- 每張任務卡對應地圖上一個寶藏格。
- 冒險者只有在完成移動並停在自己任務卡對應的寶藏格時，才取得揭露機會。
- 揭露是選擇性的；不揭露可以避免向提燈怪暴露當下位置。
- 揭露後，該任務立即完成，卡片改為全員可見。
- 若本次選擇不揭露，不能在下一回合開始時原地揭露。
- 必須在之後的回合完成移動並再次停在該寶藏格，才會重新取得揭露機會。
- 在一次移動中離開後繞回相同寶藏格，只要最後停在該格，也算再次抵達。
- 途中經過寶藏不會觸發揭露。

### 2.10 提燈怪正常回合

提燈怪骰六面為：

```text
1、1、2、2、3、3
```

- 所有冒險者完成本輪回合後，提燈怪擲一次提燈怪骰。
- 提燈怪骰每個正常回合只能擲一次，不能重擲；官方規則的重擲權只適用於冒險者骰。
- 最大移動步數為「提燈怪骰點數＋目前鎖定的冒險者骰數量」。
- 網站版允許提燈怪移動 `0～N` 步，可以不移動或提早停止。
- 每一步只能上下左右移動，不能穿牆。
- 提燈怪離開初始地牢後，不能進入入口或地牢。
- 提燈怪可以轉彎、往返或再次經過相同道路格。
- 正常回合使用過的鎖定骰不會自動解鎖，仍會保留到未來某名冒險者選擇解鎖。

### 2.11 抓捕與生命

- 只有提燈怪在自己的正常回合或插入回合主動進入冒險者所在格時，才會抓捕。
- 冒險者不能主動進入或穿越提燈怪所在格。
- 提燈怪一進入冒險者所在格，移動立即結束。
- 被捕冒險者失去 1 點生命並移到地牢。
- 提燈怪停留在發生抓捕的道路格，不會跟著進入地牢。
- 一般道路格不允許多名冒險者主動停在同格；若因特殊狀態或資料恢復而有多名冒險者同格，提燈怪進入時會一次抓捕該格全部冒險者。
- 冒險者生命降為 0 時立即出局，之後不再取得回合。
- 地牢中的冒險者若仍有生命，下一次自己的回合正常從地牢出發。

### 2.12 勝利條件

#### 冒險者勝利

- 3 至 5 人遊戲：任一冒險者揭露自己的第 5 張任務卡，該冒險者立即單獨獲勝。
- 2 人遊戲：冒險者玩家以兩枚棋子共同揭露全部 10 張任務卡，立即獲勝。

#### 提燈怪勝利

| 真人玩家 | 冒險者棋子／玩家 | 提燈怪所需生命標記 |
| --- | --- | ---: |
| 2 | 2 枚棋子，由同一人控制 | 3 |
| 3 | 2 名冒險者 | 4 |
| 4 | 3 名冒險者 | 6 |
| 5 | 4 名冒險者 | 7 |

- 提燈怪取得所需數量的生命標記時立即獲勝。
- 若勝利發生在插入回合，遊戲立即結束，觸發解鎖的冒險者不再擲骰。

## 3. 資訊公開與玩家視角

### 3.1 冒險者可見資訊

- 所有冒險者與提燈怪的目前位置。
- 所有玩家剩餘生命與已完成任務。
- 冒險者骰的完整擲骰結果與鎖定狀態。
- 自己手中的完整任務卡。
- 其他冒險者剩餘任務卡的顏色／數量，但不能看到寶藏圖案或 ID。

### 3.2 提燈怪可見資訊

- 自己的目前位置與可移動步數。
- 所有冒險者剩餘生命。
- 鎖定骰數量。
- 冒險者最後公開選用的骰子結果。
- 冒險者剩餘任務卡的顏色／數量。
- 已揭露任務卡的完整內容，以及揭露發生時對應的寶藏格。
- 不得收到任何冒險者的即時位置、完整擲骰結果、未公開任務 ID 或可推導其位置的合法移動提示。

### 3.3 公開行動記錄

- 冒險者開始與結束回合。
- 解鎖數量與提燈怪插入回合開始。
- 冒險者公開選用的骰面，但不向提燈怪公開其他骰面。
- 任務揭露、抓捕、失去生命、出局與勝負結果。
- 提燈怪的實際移動路徑屬於公開資訊，冒險者與提燈怪都能看到。

## 4. 版圖與寶藏資料

### 4.1 座標

- 版圖視覺範圍為 10 欄 × 7 列。
- 左上角為 `(1,1)`。
- `x` 向右增加，`y` 向下增加。
- 一般道路只允許上下左右相鄰格通行。
- 牆壁與通道必須由結構化鄰接資料驗證為雙向。

### 4.2 特殊位置

| 類型 | 標記位置 | 離開後第一格 |
| --- | --- | --- |
| 入口 | `(4,7)` | `(3,7)` |
| 地牢 | `(6,7)` | `(6,6)` 或 `(7,7)` |

### 4.3 寶藏分布

寶藏名稱綁定 A 至 E 五個固定類別；ID 與座標是穩定的程式識別值。

| 顏色 | ID 與座標 |
| --- | --- |
| 藍 A・黃金渡渡鳥聖像 | A1 `(1,4)`、A2 `(2,5)`、A3 `(3,4)`、A4 `(1,6)`、A5 `(4,5)` |
| 綠 B・龍眼 | B1 `(2,1)`、B2 `(3,1)`、B3 `(1,2)`、B4 `(2,3)` |

總計 23 個寶藏：藍 5、綠 4、黃 4、粉 5、紅 5。

## 5. Server 遊戲狀態機

Server 是隨機結果、合法移動、秘密資訊與勝負判定的唯一權威。Client 只能提交動作意圖，不得提交自行決定的骰面、抓捕結果或任務完成結果。

### 5.1 建議階段

| Phase | 行動者 | 合法出口 |
| --- | --- | --- |
| `adventurer_turn_start` | 當前冒險者 | 保留鎖定骰、解鎖 |
| `adventurer_forced_skip` | 當前冒險者 | 略過回合 |
| `mummy_interlude_move` | 提燈怪 | 移動一步、提早停止、抓捕後結束 |
| `adventurer_roll` | 當前冒險者 | 首擲、重擲、選擇合法骰面 |
| `adventurer_numeric_move` | 當前冒險者 | 提交完整合法路徑 |
| `adventurer_arrow_move` | 當前冒險者 | 選擇合法方向，Server 推導終點 |
| `treasure_decision` | 當前冒險者 | 揭露對應任務、暫不揭露 |
| `mummy_normal_roll` | 提燈怪 | 由 Server 擲提燈怪骰 |
| `mummy_normal_move` | 提燈怪 | 移動一步、提早停止、抓捕後結束 |
| `game_over` | 無 | 房主返回大廳或依共用框架開始新局 |

### 5.2 冒險者回合開始

Server 依序執行：

1. 若玩家已出局，跳到下一名冒險者或提燈怪正常回合。
2. 計算 `1、2、3、4、箭頭` 是否至少存在一種合法移動。
3. 若全部不合法，公開記錄原因並進入 `adventurer_forced_skip`；鎖定骰保持不變，等待當前冒險者按下「略過回合」。
4. 若五顆骰子全部鎖定且當前冒險者可行動，強制進入解鎖插入回合。
5. 若有 1 至 4 顆鎖定骰，提供「解鎖全部」或「保留」選項。
6. 若沒有鎖定骰，直接進入擲骰階段。

### 5.3 冒險者擲骰狀態

Server 保存：

- 五顆骰子的唯一 ID。
- 哪些骰子已鎖定。
- 最新一次未鎖定骰結果。
- 當前玩家可選擇的合法骰子與對應合法行動。

每次首擲或重擲：

1. Server 為所有未鎖定骰產生新結果。
2. 將新出現的提燈怪面加入鎖定集合。
3. 若鎖定數達 5，立即結束冒險者回合並安排強制解鎖。
4. 否則計算每顆非提燈怪骰是否至少存在一個合法移動。
5. Client 只能選擇 Server 回傳的合法骰子，或要求重擲全部未鎖定骰。

### 5.4 數字移動驗證

Client 提交完整座標路徑，Server 驗證：

1. 路徑步數等於所選點數。
2. 每一步都是合法的上下左右通道。
3. 沒有穿牆、越界、進入提燈怪格、入口或地牢。
4. 中途可以經過其他冒險者。
5. 終點沒有其他冒險者。
6. 同一格可以在路徑中重複出現。

### 5.5 箭頭移動驗證

Client 只提交方向，Server 根據當前狀態推導完整路徑與終點：

1. 方向必須為上、下、左、右。
2. 相鄰格必須不是障礙物。
3. 持續沿相同方向移動，直到下一格為障礙物。
4. 最後停在障礙物前一格。
5. 不接受 Client 自行指定箭頭終點。

### 5.6 提燈怪移動驗證

- Server 只向提燈怪提供從目前位置可走的相鄰道路，不提供任何會洩漏冒險者位置的預覽。
- 提燈怪每次提交一步，Server 驗證通道並更新位置。
- 每走一步後，Server 在秘密狀態檢查該格是否有冒險者。
- 若發生抓捕，立即結束提燈怪移動，不再接受剩餘步數。
- 未抓捕且仍有步數時，提燈怪可以繼續移動或選擇停止。
- 不得因某方向按鈕是否 disabled、hover 文案或預覽終點洩漏隱藏冒險者。

### 5.7 任務揭露

只有當前冒險者完成移動且終點對應其未完成任務時，Server 才建立 `treasure_decision`：

- `reveal`：公開卡片、標記完成並立即檢查冒險者勝利。
- `decline`：保留卡片，結束這次揭露機會。

Client 不能直接提交任務 ID 完成任意手牌；可揭露清單必須由 Server 產生。

### 5.8 抓捕與勝負順序

每次抓捕依序處理：

1. 提燈怪停止移動。
2. 該格每名被捕冒險者各失去 1 點生命。
3. 每名被捕冒險者各讓提燈怪取得 1 個生命標記。
4. 生命為 0 的冒險者出局；其餘被捕者移到地牢。
5. 全部抓捕結算後檢查提燈怪勝利門檻；達成時進入 `game_over`。
6. 若抓捕發生於插入回合且觸發解鎖的冒險者仍在遊戲，從地牢回到該冒險者的擲骰階段。

## 6. Server View 與隱私

### 6.1 公開 View

可包含：

- Phase、目前行動者、座位順序與在線狀態。
- 每名冒險者的生命、是否出局、已完成任務與剩餘任務顏色／數量。
- 提燈怪位置。
- 鎖定骰數量、已公開選用的冒險者骰面，以及本次提燈怪黑骰結果。
- 公開行動記錄與勝負結果。

### 6.2 冒險者私人 View

可包含：

- 自己的完整手牌。
- 所有冒險者的目前位置。
- 冒險者骰完整結果。
- Server 計算的合法骰子、合法路徑起點與待處理揭露選項。

### 6.3 提燈怪私人 View

可包含：

- 本回合剩餘步數。
- 目前可走的相鄰通道。
- 已揭露任務與其寶藏位置。

不得包含：

- 冒險者目前或歷史移動路徑。
- 冒險者目前位置。
- 未公開任務 ID。
- 冒險者未選用的骰面。
- 任何因合法目標清單差異而推導冒險者位置的資料。

## 7. 前端呈現需求

- 使用文字、色塊、矩形格線與簡單棋子標記呈現，不依賴背景圖片。
- 不加入聲音或持續動畫。
- 正式遊戲棋格不常駐顯示座標文字；座標只保留在地圖編輯器作為製圖輔助。遊戲棋格應優先呈現棋子、寶藏與合法移動狀態，座標僅能放在無障礙名稱或需要時的輔助提示。
- 抓捕使用 Lightbox 與短暫、低負擔的簡單特效；特效不能延遲 Server 狀態結算。
- 主遊戲區必須顯示地圖、目前回合、可用／鎖定骰、生命、任務進度與必要操作。
- 提燈怪正常回合擲出的黑骰結果必須公開顯示給所有玩家，直到該次移動結束。
- 不提供可回看的完整遊戲記錄分頁。主畫面只保留最近三則即時行動回饋；已揭露任務、生命、剩餘任務顏色與目前棋子位置仍持續顯示，因為它們在實體桌面上本來就是公開狀態。
- 冒險者必須有私人手牌區；提燈怪不得在 DOM、HTML attribute、預載 JSON 或前端記憶體取得秘密任務與冒險者位置。
- 冒險者 hover 或鍵盤 focus 自己尚未完成的任務卡時，棋盤暫時高亮對應寶藏格；提示只使用該玩家 Server View 已包含的私人手牌。
- 任務揭露完成後，該寶藏標記立即從所有玩家的棋盤移除，已揭露卡不再提供位置高亮。
- 數字骰移動應只允許選擇 Server 判定可完成的路徑。
- 數字路徑的逐格預覽只保留在當前冒險者的本機畫面；走滿點數並按下確認後，才一次提交完整路徑。提燈怪只能知道官方規則要求公開的所選骰面，不能收到預覽步數、方向或中途路徑。
- 箭頭骰選擇方向後由 Server 決定終點，前端只呈現結果。
- 提燈怪移動採逐步操作，並提供明確的「停止移動」指令。
- 規則面板至少包含本文件第 2 章；兩人特殊規則與資訊隱藏規則不可省略。

## 8. 必要測試案例

### 8.1 骰子與解鎖

- 1 至 4 顆鎖定骰可選擇保留或全部解鎖。
- 不能部分解鎖。
- 擲骰後不能在同一回合返回解鎖。
- 每次重擲會重擲所有未鎖定骰。
- 五顆全部鎖定時公開原因，當前冒險者按下「略過回合」後才結束回合，下一名可行動冒險者強制解鎖。
- 本輪最後一名冒險者鎖滿五顆時，正常提燈怪回合先於 5 步插入回合。
- 正常提燈怪回合結束後鎖定骰仍存在。

### 8.2 移動

- 數字骰可轉彎、往返與重複經過同格。
- 數字骰可穿越其他冒險者，但不能停在同格。
- 數字骰不能穿越提燈怪或牆壁。
- 箭頭遇到其他冒險者、提燈怪、牆、入口或地牢時停在前一格。
- 箭頭不能轉彎或移動 0 格。
- 無法完成點數的骰子不能被選用。
- 完全封鎖者只能按下「略過回合」，不能解鎖、擲骰或揭露；按下前不得自動推進。
- 從入口與地牢離開時，第一格與計步正確。

### 8.3 任務、抓捕與勝負

- 經過寶藏不觸發揭露。
- 停在寶藏可揭露或拒絕。
- 拒絕後不能原地於下一回合直接揭露。
- 提燈怪進入冒險者格立即抓捕並停止。
- 非預期有多名冒險者同格時，一次抓捕全部並逐人計算生命標記。
- 冒險者不能主動進入提燈怪格。
- 插入回合抓到觸發者後，未出局者從地牢繼續自己的回合。
- 冒險者與提燈怪達成門檻時立即結束，不接受後續行動。
- 2 人遊戲使用兩枚冒險者棋子、10 張共用任務與 3 點提燈怪勝利門檻。

### 8.4 隱私

- 提燈怪 View、WebSocket payload 與 DOM 不含冒險者位置。
- 提燈怪只收到冒險者選用的骰面，不收到完整骰面。
- 其他冒險者不能看到非自己的任務 ID。
- 偽造路徑、骰面、任務 ID、抓捕結果或剩餘步數都會被 Server 拒絕。
- 重新連線後依角色恢復正確 View，不會短暫下發其他角色的秘密資料。

## 9. 來源與網站版裁定

規則核對來源：

- [Ravensburger《Fluch der Mumie》正式規則書](https://www.ravensburger.de/spielanleitungen/ecm/Spielanleitungen/Fluch%20der%20Mumie.pdf)
- [《Gangsi 古墓迷蹤》開箱與元件說明](https://boardgame-record.blogspot.com/2016/09/gangsi.html)
- [《Gangsi》繁體中文版規則介紹](https://punchboardgame.pixnet.net/blog/posts/9577710024)

已明確採用的網站版裁定：

1. 提燈怪正常回合與插入回合的步數都是最大值，可移動 `0～N` 步並提早停止。
2. 冒險者可以選擇不揭露已抵達的任務寶藏，但之後必須完成另一次移動並再次停在該格才能揭露。
3. 入口標記為 `(4,7)`，從入口移動的第一格為 `(3,7)`。
4. 地牢為 `(6,7)`，離開地牢的第一格為 `(6,6)` 或 `(7,7)`。

## 10. 獵殺模式

### 10.1 大廳與地圖

- 3～5 名玩家，固定 1 名提燈怪及 2～4 名冒險者；每名冒險者只控制一枚棋子。
- 冒險者職業不可重複。提燈怪類型由本人選擇，大廳只向其他玩家顯示「已選擇」，開始遊戲後公開。
- 地圖可選擇性提供 `hunt.mechanisms.A/B` 兩個座標。經典模式完全忽略這些位置。
- 獵殺模式要求兩座機關完整、互不重疊、與寶藏／入口／地牢分離，且各自至少有一個可達的相鄰道路。
- 機關完成前是障礙物，冒險者只能從相鄰道路操作；完成後同一格轉為逃生出口，只有冒險者能移入並逃脫。

### 10.2 合作目標與結算

- 全隊寶藏門檻為「初始冒險者人數 × 2 + 1」。揭露只增加團隊進度，不觸發個人勝利。
- 達標後，人類可在回合開始消耗整回合操作相鄰機關，並擲一顆六面為 `X、0、1、1、1、2` 的機關骰。數字面增加對應進度；`X` 增加 1 點，並封印該機關 1 個冒險者回合。每座機關累積 3 點後，機關格直接轉為出口；不同玩家可跨回合共同累積。
- 被封印的機關在下一個冒險者完整回合內不能操作，另一座未封印機關仍可操作。提燈怪正常回合與插入回合不消耗封印；若 `X` 的進度直接使機關達到 3 點，立即開啟出口且不套用封印。
- 人類移入開啟出口後立即逃脫、位置清除並退出回合順序。另一座未完成機關仍可繼續操作。
- 所有人類都死亡時提燈怪完全勝利。遊戲只在每名人類都死亡或逃脫後結束；至少一人逃脫即代表人類達成逃生。
- 結算保存每名冒險者的 `escaped` 或 `dead` 結果。

### 10.3 最後生還者密道

- 只剩一名未死亡、未逃脫的人類時，Server 從該玩家可達且未被棋子占用的道路隨機開啟密道；寶藏格也可成為密道位置。
- 密道位置向所有人公開。人類移入立即逃脫；提燈怪移入則立即關閉密道、停止移動並結束回合。
- 提燈怪關閉密道時，機關 A、B 立即轉為逃生出口、不要求機關進度，並清除既有機關封印。

### 10.4 冒險者職業

- **騎士**：在正常回合開始，消耗整回合守護自身周圍 8 格內一名尚未受守護的其他隊友；範圍採純座標判斷，包含斜角且不受牆壁影響。守護持續到抵銷一次生命損失或受傷；擋下直接抓捕仍送入地牢。冷卻 5 個自己的正常回合。
- **工程師**：操作機關時照常擲機關骰，骰面結算後額外增加 1 點進度；擲出 `X` 仍會封印機關 1 個冒險者回合。
- **醫生**：初始生命及生命上限為 4。
- **魔法師**：正常回合開始，僅在鎖定 2～4 顆怪物骰時可免費解鎖其中一顆，之後仍可行動；每個自己的正常回合最多使用一次，不觸發插入回合，全局 3 次。五骰全鎖的強制流程不能使用。

### 10.5 受傷

- 陷阱與飛刀只造成受傷，不扣生命。
- 受傷者下一個正常回合會停用一顆當時未鎖定的骰子。停用骰不是怪物骰、不增加提燈怪步數，也不影響其他玩家。
- 該正常回合結束後效果消失；同回合再次受傷會保留到下一個正常回合。
- 騎士守護可以完整抵銷一次受傷。

### 10.6 提燈怪類型

- **陷阱鬼**：正常回合擲骰前，可在相鄰空道路免費放置或回收一個陷阱，操作後仍可擲骰。最多 2 個，放置冷卻 2 回合，回收不啟動冷卻。陷阱不得放在入口、地牢或任何地牢出口第一格；地牢出口再外一層的道路仍可放置，讓冒險者離開特殊區域後保有路線選擇。人類看不見放置資訊；任何人移入即觸發並移除。人類停步並受傷；陷阱鬼停步並立即結束回合。
- **隱形鬼**：無冷卻。可在擲骰前隱形並繼續行動，狀態跨回合維持；隱形期間不能抓捕。選擇現形時立即結束回合。人類合法路徑忽略隱形鬼；任一方移動碰撞時，移動者停在碰撞前一格，鬼在原位置向所有人現形，該移動回合結束且不抓捕、不受傷。
- **飛刀手**：正常回合可選擇上下左右投擲飛刀；投擲後立即結束回合，不能再擲骰或移動。射程不限、冷卻 2 回合，遇牆壁、封閉格、機關或關閉出口停止；第一名被命中的人類受傷。公開命中座標但不公開玩家身分或職業。
- 主動能力不能在插入回合使用。冷卻 N 回合代表後續 N 個自己的正常回合都不可使用，第 N 回合結束後恢復。

### 10.7 結算成果

- 結算逐一顯示每名冒險者的玩家名稱、職業、完成任務數、操作機關次數及逃生／死亡結果。無論機關骰面與工程師加成為何，每次操作都只計為 1 次操作貢獻。
- 提燈怪顯示玩家名稱、鬼種類及能力觸發次數。
- 陷阱鬼只在陷阱實際被冒險者或自己踩中時增加次數；放置、回收或因密道移除都不計。
- 隱形鬼只在自己撞到冒險者，或被冒險者撞到而現形時增加次數；主動現形不計。
- 飛刀手只在飛刀實際命中冒險者時增加次數；未命中不計，守護抵銷受傷仍算命中。

### 10.8 追蹤與 Server View

- 全隊完成團隊寶藏目標後，Server 啟動固定 3 個提燈怪正常回合倒數；若提燈怪在團隊目標完成前關閉密道，則從關閉密道後啟動。插入回合不遞減也不觸發。
- 地圖上方固定顯示公開倒數；第 3 個提燈怪正常回合觸發時，所有玩家收到「位置已暴露」提示，只有提燈怪 View 在該完整正常回合包含人類精確位置。回合結束後重新從 3 回合計算。
- 提燈怪平時看不到人類位置、手牌、每人任務進度、守護或受傷狀態；騎士使用守護時只公開能力已使用，不公開守護目標。
- 寶藏揭露向提燈怪公開寶藏 ID 與原始位置；機關互動公開機關及進度；飛刀公開命中座標。上述事件都不公開人類身分。
- 陷阱座標只存在陷阱鬼 View；隱形鬼座標不下發給人類 View，隱形移動的公開紀錄也不包含座標。

### 10.9 獵殺模式回合階段與操作介面

- 冒險者正常回合固定使用 `adventurer_prepare → adventurer_roll → adventurer_action → adventurer_end`。數字與箭頭行動保存在 `actionState`，不再拆成不同 Server phase。
- 提燈怪正常回合固定使用 `monster_prepare → monster_roll → monster_action → monster_end`；不需確認的擲骰與結束階段可由 Server 在同一動作內自動推進。
- 提燈怪插入回合獨立使用 `monster_interrupt_prepare → monster_interrupt_action → monster_interrupt_end`，不擲提燈怪骰、不開放特殊能力，也不遞減正常回合冷卻。
- 每個 phase 都有 Server action 白名單；前端是否顯示按鈕不影響 Server 驗證，跨階段 action 一律拒絕。
- 冒險者選擇解鎖全部骰子時，Server 保存 `resumeState.playerId` 與返回的 `adventurer_prepare`。插入回合結束後返回原玩家，不重設已使用的準備階段能力，也不退還已消耗資源；魔法師解鎖術不觸發插入回合。
- 寶藏選擇與機關結果都使用 `adventurer_end`。寶藏揭露或略過後自動換人；機關 `endState` 由 Server 保存操作者、機關、骰面、骰面進度、職業加成、理論增加、實際增加、最終進度與封印結果，只有操作者能按下「結束回合」。重連只重新呈現既有結果，不重新擲骰。
- 提燈怪 View 的 `currentPlayerId` 與當下骰面屬於公開資訊，因此操作當下看得出是誰的回合；機關 `endState` 仍移除 `operatorPlayerId`，紀錄也不附帶姓名或職業，回合結束後不提供額外的操作者身分供事後查閱。
- 沒有任何玩家輸入需求的結束流程由 Server 直接結算並換人；強制略過、移動中斷、逃生與消耗整回合的職業能力都走同一個結束結算點，避免冷卻或封印重複扣除。
- 騎士冷卻只在騎士自己的正常冒險者回合結束時遞減；提燈怪能力冷卻只在自己的正常回合結束時遞減。狀態保存建立回合 ID，因此使用當回合與插入回合都不會立即遞減。
- 機關封印由兩座機關分別保存；操作者的結束階段不消耗，下一名實際冒險者在完整回合內不能操作，該回合結束後解除。提燈怪回合不計，強制略過與提前中止仍計。
- 準備階段第一層按鈕依序為「擲冒險者骰｜解鎖全部骰子／職業能力｜操作機關」；不可用按鈕不顯示，其餘按鈕依原順序補位。騎士與雙機關選擇會進入取代第一層的二層選單。
- 擲骰階段只顯示「重擲未鎖定骰」；合法骰子直接在骰區選取。飛刀手第二層以十字排列上下左右方向，選擇後立即投擲。
- 提燈怪正常回合的骰區只顯示鎖定怪物骰與一顆自己的提燈怪骰；尚未擲骰時自己的骰面顯示 `?`。插入回合只顯示鎖定怪物骰，不顯示完整五顆冒險者骰。
