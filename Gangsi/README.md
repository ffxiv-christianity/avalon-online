# Gangsi

Gangsi 是古墓迷蹤的文字／棋盤式線上實作。房間沿用全站 Shared shell，主遊戲使用結構化矩形地圖、文字寶藏、圓形玩家棋子與提燈怪棋子，不依賴原版圖片、聲音或連續動畫。遊戲分為尊重原版的經典模式，以及合作逃生的獵殺模式。

## 模組

- `game.js`：房間、玩家、準備、角色、d100 順序、地圖選擇、Server View 與即時 action 入口。
- `engine.js`：經典模式引擎及模式分派入口。
- `hunt-engine.js`：獵殺模式的合作目標、職業、提燈怪能力、機關、出口、密道、追蹤與資訊遮蔽。
- `map-format.js`：地圖 schema、尺寸、座標、牆壁、特殊區域、寶藏 ID 與驗證。
- `map-classes.js`：地圖格 class、固定屬性與互動定義。
- `map-catalog.js`：載入 `maps/index.json` 及內建地圖。
- `public/map-editor/map-editor.js`：以同一份 schema 編輯並匯出地圖 JSON。
- `rule.md`：遊戲邏輯與前端規則視窗的來源。
- `能力checklist.md`：新增或修改職業／提燈怪能力時必須複製並填寫的驗收紀錄。

地圖資料與棋子規則分離。更換地圖只改變格子、通行邊、入口、地牢、寶藏及可選的獵殺標記；冒險者及提燈怪仍透過相同 class 屬性與規則互動。

## 地圖 catalog

- `maps/index.json` 是準備大廳可選地圖清單。
- `maps/classic.json` 是依目前確認資料建立的原版配置。
- `maps/test-map.json` 是地圖編輯器匯出的測試地圖。
- 新地圖由編輯器匯出後放入 `maps/`，再登錄到 `maps/index.json`；伺服器啟動時會驗證 catalog 及每份地圖。
- 勾選「隨機地圖」時，大廳不公開實際選擇；按下開始後才由伺服器抽選並揭露。
- `hunt.mechanisms.A/B` 是向後相容的可選資料；經典模式完全忽略，獵殺模式中每座機關完成後會在同一格轉為逃生出口。
- 沒有完整獵殺標記的地圖仍列在大廳，但獵殺模式會標為不可用，也不會被隨機選中。

地圖限制為 6–14 欄、5–12 列。實際遊戲不在每格常駐顯示座標，座標只保留在編輯器與棋格的無障礙名稱。

## 房間流程

房主先選擇經典或獵殺模式。經典模式維持 2～5 人與原版勝負；獵殺模式限制 3～5 人，每名冒險者只控制一枚棋子，職業不可重複，提燈怪類型在遊戲開始時公開。

1. `lobby`：玩家自行選擇冒險者或提燈怪、冒險者填一字棋子、擲 d100、準備；全房必須恰有一名提燈怪。
2. 冒險者依 d100 由高至低行動；兩人局由唯一冒險者控制兩顆棋子。
3. 冒險者可擲骰、選用數字或箭頭骰、移動、保留鎖骰或一次解鎖全部鎖骰。
4. 解鎖會讓提燈怪選擇是否執行插入回合；五顆骰全部鎖定時依規則強制安排插入回合。
5. 提燈怪正常回合先擲黑骰，再加上鎖定骰數得到最多步數，也可提早停止。
6. 冒險者完成移動到自己的寶藏時可選擇揭露或暫不揭露；拒絕後必須先移動並再次到達才可重新選擇。
7. 提燈怪主動進入冒險者所在格時抓捕該格所有冒險者，扣生命並送入地牢或淘汰。
8. 任一方達成條件後進入 `game_over`，房主可返回大廳開始下一局。

## Server View

- 冒險者可看到冒險者棋子位置、完整冒險者骰與自己的任務手牌。
- 提燈怪看不到任何冒險者位置、未揭露任務 ID 或冒險者骰面，只看到鎖骰數、最後公開使用骰面、各玩家 A–E 類任務剩餘數量與已揭露任務。
- 合法骰子、數字路徑、箭頭方向與提燈怪下一格都由伺服器提供；前端只負責呈現與送回選擇。
- 記錄、錯誤訊息與合法目標也遵守相同隱私邊界，避免從旁通道推回隱藏位置。
- 獵殺模式的提燈怪看不到寶藏揭露者、機關操作者、受傷者狀態、隱藏陷阱以外的私人資料；只有位置追蹤生效的正常回合會收到冒險者座標。
- 隱形鬼的座標會從冒險者 View 移除；人類合法路徑不會因隱形鬼而旁洩其所在格。
- 冒險者 View 不包含提燈怪能力冷卻；能力冷卻只向提燈怪本人顯示。

## 能力開發

新增或修改冒險者職業、提燈怪類型前，先複製並填寫 [`能力checklist.md`](./能力checklist.md) 的完整能力紀錄。回合結束時機、每回合限制、Server View 隱私及固定測試都是必填項目，不能只依賴前端隱藏或隨機模擬。

## UI 契約

Gangsi 採用 `COMMON_ROOM_FRAMEWORK.md` 的 Board Game Template：保留玩家矩陣、手牌、行動資訊、控制列與右上角回合提示，但移除桌機四張 `status-strip`，把主空間留給棋盤。生命、任務、鎖骰與提燈怪得分改在玩家矩陣及控制區呈現。

地圖編輯器位於 `/Gangsi/map-editor/`，返回按鈕回到 `/Gangsi/`。規則使用房間內 overlay，不開新視窗。
