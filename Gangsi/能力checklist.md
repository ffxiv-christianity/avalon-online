# Gangsi 能力 Checklist

新增或修改冒險者職業、提燈怪類型時，必須為該能力複製一份「能力紀錄模板」，逐項填寫後才可交付。不能只勾選共用清單，也不能以隨機模擬取代能力的固定案例。

現有能力：騎士、工程師、醫生、魔法師、陷阱鬼、隱形鬼、飛刀手。

## 能力紀錄模板

### 基本資料

- [ ] 能力名稱：
- [ ] 所屬陣營與類型：冒險者職業／提燈怪類型
- [ ] 建立或修改日期：
- [ ] 規則目的與預期玩法：

### 回合與狀態

- [ ] 明確列出可發動 phase、合法角色及合法目標。
- [ ] 明確列出不能發動的 phase，包含插入回合、強制略過及遊戲結束。
- [ ] 填寫發動後的 phase：留在能力選擇／進入擲骰／進入移動／立即結束回合。
- [ ] 填寫是否消耗整個回合，以及命中、失敗、無目標時是否同樣結束回合。
- [ ] 填寫每回合使用上限、全局次數、冷卻回合及開始／遞減／重置時機。
- [ ] 填寫重複發動、效果疊加、斷線重連及回到地牢時的處理。
- [ ] 為新增狀態指定唯一欄位，不借用語意不同的冷卻或暫存欄位。

### 地圖與互動

- [ ] 列出可互動的格子 class、距離、牆壁、封閉格、機關、出口、密道及寶藏規則。
- [ ] 列出與其他棋子、陷阱、隱形、受傷、守護、抓捕及逃脫的先後順序。
- [ ] 合法目標由 Server 計算並下發；前端不得自行擴張合法範圍。
- [ ] 不同地圖只改變資料，不得改變能力與 class 的互動規則。

### 資訊與隱私

- [ ] 分別列出能力使用者、隊友、對手可見的狀態、位置、冷卻、剩餘次數與結果。
- [ ] 檢查 Server View 已移除不應下發的欄位，不只在前端隱藏文字。
- [ ] 檢查 `legal`、錯誤訊息、公開紀錄、私人紀錄及 hover 提示不會旁洩資訊。
- [ ] 填寫命中、失敗與無效果時的公開紀錄內容。

### 前端流程

- [ ] 第一個按鈕描述玩家現在要做的事，不使用「繼續」或「進入下一階段」代替具體動作。
- [ ] 需要方向或目標時，先顯示能力按鈕，選擇後才展開目標，並提供取消出口。
- [ ] 顯示正確的行動階段、可用／冷卻／已用完狀態及剩餘次數。
- [ ] 發動後 UI 立即收起，不能保留上一回合的展開或選取狀態。
- [ ] 按鈕、提示與狀態文字在桌機和行動版不溢出、不遮擋棋盤。

### 文件與測試

- [ ] `rule.md` 與 `public/index.html` 已同步發動時機、效果、限制、冷卻及回合是否結束。
- [ ] 固定測試涵蓋合法發動、錯誤角色、錯誤 phase、錯誤目標及資源不足。
- [ ] 固定測試涵蓋每回合上限、全局次數、冷卻的第一回合與恢復回合。
- [ ] 固定測試涵蓋發動後 phase、下一位玩家及命中／失敗／無目標分支。
- [ ] Server View 測試分別驗證能力使用者與對手實際收到的欄位。
- [ ] 前端靜態契約或互動測試涵蓋按鈕、展開、取消及狀態重置。
- [ ] 能力納入獵殺模式 10 次隨機完整遊戲測試，且不以寫死結果通過。
- [ ] 已執行 Gangsi 完整測試。
- [ ] 已依 `COMMON_ROOM_FRAMEWORK.md` 詢問本次是否執行 Visual QA，並記錄使用者決定與結果：
