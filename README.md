# 阿瓦隆線上主持工具

https://avalon-online-lhem.onrender.com/

這是阿瓦隆線上多人即時版本，由系統擔任主持人，讓所有朋友都可以一起當玩家。

專案使用 Node.js + WebSocket 同步房間狀態，可部署到 Render。前端直接由同一個 Node.js 服務提供。

## 主要功能

- 房間代碼與邀請連結
- 每位玩家使用自己的裝置加入
- 支援 4 到 10 人遊玩
- 房主可設定遊戲人數、牌庫、領袖規則、每輪任務人數
- 各人數推薦牌庫一鍵套用
- 玩家進房後擲 d100，且每局只能擲一次
- 開始遊戲後依 d100 由大到小排序
- 每位玩家準備，全部準備後才可由房主開始遊戲
- 投票進度即時同步，例如 `0 / 5` 到 `5 / 5`
- 全員投完後公開明票
- 任務牌匿名提交，只公開成功/失敗與失敗牌數
- 領袖指示物、退役領袖徽章、職業圖示
- 指定領袖模式下，已有退役領袖徽章者不可再被指定
- 投票失敗時照順時針輪替，未出任務的領袖不會拿退役領袖徽章
- 可選王者之劍擴充，可設定是否允許同一玩家重複持有；持劍者可公開選擇發動或不發動
- 可選湖中女神擴充，建議大於七人遊戲使用；初始由擲骰第二大的玩家持有，系統會私下顯示查驗結果並傳遞指示物
- 打字聊天
- 投票結果、任務結果、遊戲結果可按 emoji 反應
- 稱號與成就系統，玩家列表 hover 可查看已取得稱號與動態次數
- 同步版本顯示，方便確認目前是否跟上房間狀態
- 房間建立後 6 小時自動清除

## 四人規則

四人局固定為 3 正義方、1 邪惡方。

四人局的梅林規則不同：

> 只能看見一個隊友身份。若最後被刺客刺殺，邪惡方勝利。

## 斷線與重連

每位玩家加入後，瀏覽器會把 `roomCode` 與 `playerId` 存在 localStorage。

- 重新整理、網頁意外關閉、短暫斷線：回到同一個房間網址，按「以某某重新連線」即可拿回原玩家 ID。
- 邀請連結第一次打開：必須輸入名字加入。
- 若遊戲已開始且玩家 ID 遺失，系統不會讓新玩家插入進行中的遊戲。
- 沒有倒數限制，投票、任務、刺殺等階段會停在目前進度等待玩家回來。
- WebSocket 斷線後前端會自動重連並重新同步完整狀態。

## 本機執行

```powershell
npm install
npm start
```

預設網址：

```text
http://localhost:4173
```

如果 4173 被占用，可以改用其他 port：

```powershell
$env:PORT=4174
npm start
```

## 測試

```powershell
npm test
```

目前測試包含：

- 房間建立、加入、重連
- 大廳設定、擲骰、準備、開局
- 四人局梅林資訊
- 投票、任務、領袖限制
- 投票失敗輪替與五次否決
- 聊天與 emoji 反應
- 刺殺與重置房間
- 稱號與成就門檻

## 部署

專案可部署到 Render Web Service。

建議設定：

- Build Command: `npm install`
- Start Command: `npm start`
- Node.js 版本：18 以上

部署後使用 Render 提供的網址即可讓玩家線上同步遊玩。
