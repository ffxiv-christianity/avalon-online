# 阿瓦隆線上主持工具

https://avalon-online-lhem.onrender.com/

這是免主持人的阿瓦隆多人即時版本。系統會負責角色分配、流程提示、投票、任務結算與私密情報，讓所有朋友都能一起當玩家。

專案使用 Node.js + WebSocket 同步房間狀態，可部署到 Render。前端直接由同一個 Node.js 服務提供，並針對手機操作與 PWA 安裝體驗最佳化。

## 主要功能

- 房間代碼與邀請連結
- 每位玩家使用自己的裝置加入
- 支援 4 到 10 人遊玩
- 手機版操作介面，玩家選擇區以雙欄排列減少垂直捲動
- 支援安裝為 PWA（仍需連線，並非離線遊戲）
- 房主可設定遊戲人數、牌庫、領袖規則、每輪任務人數
- 房主可將房主權限轉移給其他玩家
- 各人數推薦牌庫一鍵套用
- 玩家進房後擲 d100，且每局只能擲一次
- 開始遊戲後依 d100 由大到小排序
- 每位玩家準備，全部準備後才可由房主開始遊戲
- 投票進度即時同步，例如 `0 / 5` 到 `5 / 5`
- 全員投完後公開明票
- 公開投票與任務結果可設定不延遲、3 秒或 5 秒後才允許領袖繼續
- 任務牌匿名提交，只公開成功/失敗與失敗牌數
- 身份辨識階段使用私密 lightbox 顯示角色與可見玩家
- 領袖指示物、退役領袖徽章、職業圖示
- 指定領袖模式下，已有退役領袖徽章者不可再被指定
- 投票失敗時照順時針輪替，未出任務的領袖不會拿退役領袖徽章
- 可選王者之劍擴充；領袖可選擇不給劍，或交給參與任務的其他玩家，但不能交給自己；持劍者選定目標並確認後，會私下看到該玩家原本的任務牌，再翻轉並公開結算
- 可選湖中女神擴充，建議大於七人遊戲使用；初始由擲骰第二大的玩家持有，選定查驗目標並確認後才會顯示私密結果，且結果不寫入遊戲記錄
- 打字聊天
- 聊天、玩家、任務與記錄使用固定資訊分頁
- 聊天有未讀提示；玩家提示只會在新玩家進入房間時出現
- 投票結果、任務結果、遊戲結果可按 emoji 反應
- 稱號與成就系統，玩家列表 hover 可查看已取得稱號與動態次數
- 同步版本顯示，方便確認目前是否跟上房間狀態
- 房間無人在線 30 分鐘後自動清除；有人在線時不會清除

## 四人規則

四人局固定為 3 正義方、1 邪惡方。

四人局的梅林規則不同：

> 只能看見一個隊友身份。若最後被刺客刺殺，邪惡方勝利。

## 斷線與重連

每位玩家加入後，瀏覽器會把 `roomCode`、`playerId` 與名稱存入 localStorage，網址也會帶有對應的 `player` ID。

- 重新整理、網頁意外關閉、短暫斷線：回到同一個房間網址，按「以某某重新連線」即可拿回原玩家 ID。
- 同一瀏覽器可保存多個玩家身分；不同分頁的玩家網址重新整理後，會各自找回原本的玩家 ID。
- 若網址沒有指定玩家，也可輸入曾使用過的名稱切換到該玩家身分。
- 邀請連結第一次打開：必須輸入名字加入。
- 若遊戲已開始且玩家 ID 遺失，系統不會讓新玩家插入進行中的遊戲。
- 沒有倒數限制，投票、任務、刺殺等階段會停在目前進度等待玩家回來。
- WebSocket 斷線後前端會自動重連並重新同步完整狀態。

## 管理統計

設定環境變數 `ADMIN_TOKEN` 後，可以用以下網址查看目前房間與在線人數：

```text
https://你的網域/admin/stats?token=你的ADMIN_TOKEN
```

統計頁不顯示玩家名稱，只顯示房間數、連線數、玩家數、在線玩家數，以及各房間的在線人數。

## 本機執行

```powershell
npm install
npm start
```

預設網址：

```text
http://localhost:4173
```

如果 4173 被占用，可以改用其他 port：

```powershell
$env:PORT=4174
npm start
```

## 測試

```powershell
npm test
```

目前測試包含：

- 房間建立、加入、重連與新玩家提示事件
- 同一瀏覽器保存多個玩家身分與舊 session 遷移
- 大廳設定、擲骰、準備、開局
- 四人局梅林資訊
- 投票、任務、領袖限制
- 擲骰、準備、投票、token 與上下線狀態不會誤觸玩家未讀提示
- 投票失敗輪替與五次否決
- 聊天與 emoji 反應
- 王者之劍與湖中女神流程及私密結果權限
- 刺殺與重置房間
- 稱號與成就門檻

## 部署

專案可部署到 Render Web Service。

建議設定：

- Build Command: `npm install`
- Start Command: `npm start`
- Node.js 版本：18 以上

部署後使用 Render 提供的網址即可讓玩家線上同步遊玩。
